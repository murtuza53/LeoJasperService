# LeoJasperService — API contract (for AI integrators)

You are integrating with LeoJasperService, a JasperReports-backed report
renderer. The contract below is the entire API surface.

## Base URL
- Development: http://localhost:8080
- Production:  whatever the user deployed at (ask them).

## Authentication
EVERY /api/* request (except /api/admin/* and /api/auth/*) MUST include:
  X-Key-Id:          (issued at company registration)
  X-Key-Secret:           (issued at company registration; shown ONCE)

The admin issues these credentials on the /license.html page. The secret is
stored on the server only as a SHA-256 hash; if it's lost, the admin must
"rotate" the company's key to issue a new one.

For elevated security, a company may opt into HMAC request signing
(hmacRequired=true). In that mode the client sends X-Key-Id, X-Timestamp,
X-Nonce, X-Signature and the secret never traverses the wire. (Currently
the HMAC verifier is a placeholder — Tier 1 secret auth is the working path.)

The admin browser uses session cookie auth (LEOJASPER_ADMIN_SID) for
/api/admin/*. Clients never need that cookie.

## License scopes
At company registration the admin picks one of:
- READ   — list / get / render only; no uploads, no deletes
- RENDER — adds /api/reports/render
- FULL   — full assets CRUD, synthesis, render

Scope is enforced server-side; a stolen READ key cannot delete assets.

## Errors
- 401 — missing/invalid X-Key-Id+Secret, or admin not logged in (registry locked)
- 403 — scope does not permit the operation
- 429 — rate-limited (default 60 req/min per key) or temp auth-lockout
- 503 — admin has not unlocked the registry yet (no admin login since restart)

## CORS
Wide-open in dev: allowedOriginPatterns="*", all methods, expose
Content-Disposition + X-Job-Id. Browser clients do not need a proxy.

## Render synchronously
POST /api/reports/render
Content-Type: multipart/form-data

Form fields:
- templateId    string. id of a server-stored template (one of GET /api/templates).
                Use this OR `template`, not both.
- template      file (.jrxml). One-off upload, NOT stored on server.
- data          file. The payload — JSON, XML, or CSV bytes.
- dataInline    string. Alternative: paste the data inline.
- inputFormat   string. "json" | "json-doc" | "xml" | "csv".
                "json-doc" expects ONE file with two top-level sections:
                  { "parameters": { ... }, "data": [ ... ] }
                The parameters object is merged into the report's parameter
                map (the `parameters` form field, if also sent, overrides
                values from the file). The data array iterates as detail rows.
- outputFormat  string (REQUIRED). One of:
                  pdf | xlsx | xls | html | png | png-zip
                  docx | odt | rtf | csv | text | pptx
- parameters    string (optional). JSON object, becomes Map
                of report parameters (used as $P{NAME} in the template).

Returns:
- 200 OK with the report bytes.
  Content-Type matches the format (application/pdf, ..., image/png, ...).
  Content-Disposition: attachment; filename="report."
- 4xx/5xx JSON body: { "status": int, "error": str, "message": str, "timestamp": iso8601 }

In a browser client: ALWAYS read the success response as a Blob (fetch:
res.blob(); axios: responseType: "blob"). NEVER JSON.parse the success body.

## Render asynchronously (for slow reports)
POST /api/reports/render-async  → 202 { "jobId": str, "status": "PENDING" }
GET  /api/reports/jobs/{id}     → ReportJob JSON (status, timestamps, error)
GET  /api/reports/jobs/{id}/result → bytes when status == "COMPLETED"
GET  /api/reports/jobs?limit=20 → recent jobs

ReportJob.status: PENDING | RUNNING | COMPLETED | FAILED | CANCELLED.
Jobs evict from the in-memory store after 2h by default.

## Templates
GET    /api/templates           → ["id1", "id2", ...]
GET    /api/templates/{id}      → .jrxml bytes
POST   /api/templates           → multipart {id, file}; replaces if exists
DELETE /api/templates/{id}      → 204 or 404

## Assets (per-company; scope FULL for writes/deletes)
GET    /api/assets                          → grouped {bucket → [{name,size,...}]}
GET    /api/assets/{bucket}                 → list one bucket
POST   /api/assets/{bucket}/{name}          → multipart "file" upload
GET    /api/assets/{bucket}/{name}          → download (?inline=true for preview)
DELETE /api/assets/{bucket}/{name}          → soft-delete (30-day restore)
GET    /api/assets/_trash                   → list this company's trash
POST   /api/assets/_trash/{trashName}/restore  → restore from trash

Buckets: images, templates, data, other. Path traversal blocked.

## Admin (admin session cookie required, NOT a license key)
POST   /api/admin/login           → {ok, mustChangePassword, mustSetAssetsRoot, hasCompanies}
POST   /api/admin/logout
POST   /api/admin/change-password → re-encrypts credentials.enc + companies.enc
POST   /api/admin/assets-root     → set the filesystem root (must be writable)
GET    /api/admin/companies       → list registered companies
POST   /api/admin/companies       → register; secret is in the response, ONCE
POST   /api/admin/companies/{keyId}/rotate   → new secret; old invalidated
POST   /api/admin/companies/{keyId}/disable  → block all calls under that keyId
POST   /api/admin/companies/{keyId}/enable
GET    /api/admin/companies/{keyId}/audit    → tamper-evident audit log
GET    /api/admin/companies/{keyId}/trash    → admin can also see/restore trash

## Health & metrics
GET /actuator/health     → {"status":"UP", ...}
GET /actuator/prometheus → Micrometer metrics (leojasper_compile_seconds, leojasper_fill_seconds{inputFormat}, leojasper_export_seconds{outputFormat}, leojasper_errors_total{stage})

## Parameter type coercion (since 2026-05-12)
Before fillReport, the service walks the JRXML's declared  list and
coerces supplied values to match the declared Java type:
  - Number → Number: silently converted. Integer 0 works for a Double param,
    BigDecimal works for a Double, a Long works for an Integer, etc.
  - Anything → String: String.valueOf(value).
  - String "true"/"false" → Boolean.
  - String "5" → Number: REJECTED with HTTP 400 and the message:
      "Parameter '' is declared as  but received
        (""). Only Number→Number and Anything→String
       coercions are applied automatically; fix the source data to send the
       correct type."
Practical effect: producers that emit "qty": 2 (int) and "qty": 2.0 (float)
both work for a java.lang.Double parameter. But "qty": "2" still fails — make
sure the source sends a JSON number when the report expects one.

## Image resolution (since 2026-05-12)
JRXML  evaluations are intercepted:
  - URLs (http://, https://): fetched with a 5 s connect+read timeout. On
    failure (DNS, 4xx/5xx, timeout) the placeholder is substituted, NOT an
    error. data: URIs are decoded by JR natively.
  - Filenames ("logo.png", "MTC_LOGO", "/Files/Branding/X.png"): basename
    looked up under the calling company's images/ bucket first, then under
    companyDir/ and the other buckets. The asset must have been uploaded
    via POST /api/assets/images/.
  - Anything not found: a generated 300×80 grey "Resource not found" PNG is
    returned so the report still renders. To make a specific image fatal on
    failure, set onErrorType="Error" on that  element in the JRXML.
The filter only applies to image-like paths (URLs, image extensions, or
extension-less identifiers). Subreports, properties files, and fonts flow
through to JR's default resolvers unchanged.

## Tips that catch every integrator the first time
1. Two JSON shapes are supported, controlled by inputFormat:
   - inputFormat="json"     → root must be an array of objects, one per row:
                                [ {"name":"A","qty":3}, {"name":"B","qty":7} ]
                              Parameters come from the multipart "parameters" field only.
   - inputFormat="json-doc" → single file with TWO sections:
                                { "parameters": {"customer.code":"C0002", ...},
                                  "data":       [ {...}, {...} ] }
                              The "parameters" object is merged into the report
                              parameter map; the "data" array iterates as rows.
                              Useful when the source system (an ERP, billing
                              backend) already produces one self-contained file
                              per document.
   To iterate a nested array under inputFormat="json", the JRXML needs
   items.
2. CSV input must include a header row matching the JRXML .
3. For grouped reports, INPUT MUST BE PRE-SORTED by the group field.
   The server does not sort.
4. Field types in the JRXML matter (java.lang.Integer rejects non-numeric
   strings). Use java.lang.String when in doubt.
5. Currency / date display is handled by the JRXML's pattern attribute
   (e.g. pattern="$#,##0.00"), NOT by client-side formatting.
6. Async mode is necessary for any report that takes >30s, since browsers
   and proxies routinely time out sync requests.
7. CORS in production should be locked down to your real frontend origin
   (WebConfig.java in the service repo).

Authentication ¶

Every /api/* request other than /api/admin/* and /api/auth/* must include both headers: 

X-Key-Id:     <publicKeyId>
X-Key-Secret: <secret>

Credentials are issued by the admin on the license page. The secret is shown exactly once at creation. To recover it, rotate the company's key (which immediately invalidates the old one).

Why two headers instead of one?

The keyId is public — safe to log, paste in URLs, include in error messages. The secret is private; it never echoes back in any response, never appears in logs, and is stored on the server only as a SHA-256 hash compared in constant time.

Scopes

ScopeAllowedTypical use
READ list, get, render Headless reporting embed; auditors
RENDER READ + render-time multipart inputs Most production printing clients
FULL everything (assets CRUD, synthesis) Operations; admin's own integrations

Scope is enforced server-side; a stolen READ key cannot delete assets.

Defenses against a stolen secret

  • TLS-only in production — the header is plaintext on the wire and nothing application-level beats HTTPS.
  • Constant-time SHA-256 compare; secret never exists in plaintext at rest.
  • Rate-limited to 60 req/min per keyId; misuse triggers 429.
  • Auth-failure lockout — 5 wrong attempts in 5 minutes locks the source IP for the rest of that window.
  • Soft-delete with 30-day retention — even with a FULL key, an attacker can't permanently destroy assets within the window.
  • Tamper-evident audit log per company in .meta/audit.jsonl; each line carries the prev-line hash.
  • Rotation — admin issues a new secret at any time; old one is invalidated immediately. keyId stays so audit history is continuous.
  • HMAC mode (Tier 2, opt-in per company) — secret never traverses the wire; client signs each request, server verifies.

Vault adoption on admin login since 2026-05-12

Strictly an admin-side concern — clients never need this. When pointing a fresh install at the assets folder from a previous version, the admin logs in with an optional vaultPath: 

POST /api/admin/login
Content-Type: application/json

{ "username":  "admin",
  "password":  "<the old admin password>",
  "vaultPath": "D:/leojasper-data" }

The path's .leojasper/credentials.enc is unlocked with the supplied password and adopted for the session. Wrong password = 401, no overwrite. Missing file at that path = 404. Path-drift correction rewrites the stored assetsRoot to whatever the admin typed. Lost old password = unrecoverable; AES-GCM has no backdoor.

Companion guarantee: setting the assets root through the UI to a path that already has a .leojasper/credentials.enc is rejected with HTTP 400 — there is no longer any way to silently overwrite a vault that contains real client secrets.

Browser quirk: image previews can't send custom headers

A browser <img src="…/api/assets/images/logo.png"/> cannot attach X-Key-Id/X-Key-Secret automatically. For preview thumbnails, fetch with fetch(url, { headers: {…} }), get a Blob, and set img.src = URL.createObjectURL(blob). The assets page does this — copy the pattern from its source if you need it.

Sync render (browser fetch)

const BASE = "http://localhost:8080";

async function renderReport({ templateId, data, inputFormat, outputFormat, parameters }) {
  const fd = new FormData();
  if (templateId) fd.append("templateId", templateId);
  if (data instanceof Blob) fd.append("data", data);
  else fd.append("dataInline", typeof data === "string" ? data : JSON.stringify(data));
  fd.append("inputFormat",  inputFormat);
  fd.append("outputFormat", outputFormat);
  if (parameters) fd.append("parameters", JSON.stringify(parameters));

  const res = await fetch(`${BASE}/api/reports/render`, { method: "POST", body: fd });
  if (!res.ok) {
    const err = await res.json().catch(() => ({ message: res.statusText }));
    throw new Error(err.message);
  }
  return await res.blob();
}

// Trigger a download in the browser
function downloadBlob(blob, filename) {
  const url = URL.createObjectURL(blob);
  const a   = Object.assign(document.createElement("a"), { href: url, download: filename });
  a.click();
  URL.revokeObjectURL(url);
}

// usage
const blob = await renderReport({
  templateId:   "sample",
  data:         [{ name: "Alpha", qty: 3 }, { name: "Bravo", qty: 7 }],
  inputFormat:  "json",
  outputFormat: "pdf"
});
downloadBlob(blob, "report.pdf");

Async render with polling

async function renderAsync(opts, { intervalMs = 1000, timeoutMs = 5*60_000 } = {}) {
  const fd = buildForm(opts);                                // same FormData builder as above
  const r1 = await fetch(`${BASE}/api/reports/render-async`,
                         { method: "POST", body: fd });
  if (!r1.ok) throw new Error((await r1.json()).message);
  const { jobId } = await r1.json();

  const deadline = Date.now() + timeoutMs;
  while (Date.now() < deadline) {
    const j = await (await fetch(`${BASE}/api/reports/jobs/${jobId}`)).json();
    if (j.status === "COMPLETED") {
      const r3 = await fetch(`${BASE}/api/reports/jobs/${jobId}/result`);
      return r3.blob();
    }
    if (j.status === "FAILED" || j.status === "CANCELLED") throw new Error(j.error);
    await new Promise(r => setTimeout(r, intervalMs));
  }
  throw new Error("Timed out");
}

Reusable hook (drop into src/hooks/useReport.ts)

import { useState, useCallback } from "react";

const BASE = import.meta.env.VITE_REPORTS_BASE_URL ?? "http://localhost:8080";

export type RenderOpts = {
  templateId?:  string;
  template?:    Blob | File;
  data?:        Blob | File | string | unknown;
  inputFormat:  "json" | "json-doc" | "xml" | "csv";
  outputFormat: "pdf" | "xlsx" | "xls" | "html" | "png" | "png-zip"
              | "docx" | "odt" | "rtf" | "csv" | "text" | "pptx";
  parameters?:  Record<string, unknown>;
};

export function useReport() {
  const [loading, setLoading] = useState(false);
  const [error,   setError]   = useState<string | null>(null);

  const render = useCallback(async (opts: RenderOpts, filename = "report") => {
    setLoading(true); setError(null);
    try {
      const fd = new FormData();
      if (opts.templateId) fd.append("templateId", opts.templateId);
      if (opts.template)   fd.append("template",   opts.template);
      if (opts.data instanceof Blob) fd.append("data", opts.data);
      else if (opts.data !== undefined) {
        fd.append("dataInline",
          typeof opts.data === "string" ? opts.data : JSON.stringify(opts.data));
      }
      fd.append("inputFormat",  opts.inputFormat);
      fd.append("outputFormat", opts.outputFormat);
      if (opts.parameters) fd.append("parameters", JSON.stringify(opts.parameters));

      const res = await fetch(`${BASE}/api/reports/render`, { method: "POST", body: fd });
      if (!res.ok) {
        const j = await res.json().catch(() => ({ message: res.statusText }));
        throw new Error(j.message);
      }
      const blob = await res.blob();
      const url  = URL.createObjectURL(blob);
      Object.assign(document.createElement("a"),
                    { href: url, download: `${filename}.${opts.outputFormat}` }).click();
      URL.revokeObjectURL(url);
    } catch (e: any) {
      setError(e.message); throw e;
    } finally {
      setLoading(false);
    }
  }, []);

  return { render, loading, error };
}

axios client wrapper

The crucial detail with axios: responseType: "blob". Without it, axios tries to JSON-parse the binary success body and corrupts it.

import axios from "axios";

const api = axios.create({
  baseURL: import.meta.env.VITE_REPORTS_BASE_URL ?? "http://localhost:8080",
});

export async function renderReport(opts: {
  templateId?: string; data: any;
  inputFormat: "json" | "json-doc" | "xml" | "csv";
  outputFormat: string;
  parameters?: Record<string, unknown>;
}): Promise<Blob> {
  const fd = new FormData();
  if (opts.templateId) fd.append("templateId", opts.templateId);
  if (opts.data instanceof Blob) fd.append("data", opts.data);
  else fd.append("dataInline",
                 typeof opts.data === "string" ? opts.data : JSON.stringify(opts.data));
  fd.append("inputFormat",  opts.inputFormat);
  fd.append("outputFormat", opts.outputFormat);
  if (opts.parameters) fd.append("parameters", JSON.stringify(opts.parameters));

  try {
    const res = await api.post("/api/reports/render", fd, { responseType: "blob" });
    return res.data;                                // already a Blob
  } catch (e: any) {
    // For non-2xx, axios still gave us the blob — read it as JSON for the message.
    if (e.response?.data instanceof Blob) {
      const text = await e.response.data.text();
      try { throw new Error(JSON.parse(text).message); }
      catch { throw new Error(text); }
    }
    throw e;
  }
}

Injectable Angular service

// reports.service.ts
import { Injectable } from "@angular/core";
import { HttpClient, HttpErrorResponse } from "@angular/common/http";
import { Observable, catchError, map, throwError } from "rxjs";

export type OutputFormat =
  "pdf" | "xlsx" | "xls" | "html" | "png" | "png-zip" |
  "docx" | "odt" | "rtf" | "csv" | "text" | "pptx";

@Injectable({ providedIn: "root" })
export class ReportsService {
  private base = "http://localhost:8080";              // or read from environment.ts

  constructor(private http: HttpClient) {}

  render(opts: {
    templateId?: string;
    data: Blob | string | unknown;
    inputFormat: "json" | "json-doc" | "xml" | "csv";
    outputFormat: OutputFormat;
    parameters?: Record<string, unknown>;
  }): Observable<Blob> {
    const fd = new FormData();
    if (opts.templateId) fd.append("templateId", opts.templateId);
    if (opts.data instanceof Blob) fd.append("data", opts.data);
    else fd.append("dataInline",
                   typeof opts.data === "string" ? opts.data : JSON.stringify(opts.data));
    fd.append("inputFormat",  opts.inputFormat);
    fd.append("outputFormat", opts.outputFormat);
    if (opts.parameters) fd.append("parameters", JSON.stringify(opts.parameters));

    return this.http
      .post(`${this.base}/api/reports/render`, fd, { responseType: "blob" })
      .pipe(catchError((err: HttpErrorResponse) => this.toJsonError(err)));
  }

  private toJsonError(err: HttpErrorResponse): Observable<never> {
    if (err.error instanceof Blob) {
      return new Observable(sub => {
        const r = new FileReader();
        r.onload = () => {
          try { sub.error(new Error(JSON.parse(r.result as string).message)); }
          catch { sub.error(new Error(r.result as string)); }
        };
        r.readAsText(err.error);
      });
    }
    return throwError(() => err);
  }
}

    templateId?: string;
    data: Blob | string | unknown;
    inputFormat: "json" | "json-doc" | "xml" | "csv";
    outputFormat: string;
    parameters?: Record<string, unknown>;
  }, filename = "report") {
    loading.value = true; error.value = null;
    try {
      const fd = new FormData();
      if (opts.templateId) fd.append("templateId", opts.templateId);
      if (opts.data instanceof Blob) fd.append("data", opts.data);
      else fd.append("dataInline",
                     typeof opts.data === "string" ? opts.data : JSON.stringify(opts.data));
      fd.append("inputFormat",  opts.inputFormat);
      fd.append("outputFormat", opts.outputFormat);
      if (opts.parameters) fd.append("parameters", JSON.stringify(opts.parameters));

      const res = await fetch(`${BASE}/api/reports/render`, { method: "POST", body: fd });
      if (!res.ok) throw new Error((await res.json()).message ?? res.statusText);
      const blob = await res.blob();
      const url  = URL.createObjectURL(blob);
      Object.assign(document.createElement("a"),
                    { href: url, download: `${filename}.${opts.outputFormat}` }).click();
      URL.revokeObjectURL(url);
    } catch (e: any) {
      error.value = e.message; throw e;
    } finally {
      loading.value = false;
    }
  }

  return { render, loading, error };
}

import json, requests
from pathlib import Path

BASE = "http://localhost:8080"

def render_report(*, template_id=None, template_path=None,
                  data=None, data_path=None,
                  input_format, output_format, parameters=None) -> bytes:
    files = {}
    if template_id:    files["templateId"]  = (None, template_id)
    if template_path:  files["template"]    = ("t.jrxml", Path(template_path).read_bytes(),
                                               "application/xml")
    if data_path:      files["data"]        = (Path(data_path).name,
                                               Path(data_path).read_bytes(),
                                               "application/octet-stream")
    elif data is not None:
        body = data if isinstance(data, str) else json.dumps(data)
        files["dataInline"] = (None, body)
    files["inputFormat"]  = (None, input_format)
    files["outputFormat"] = (None, output_format)
    if parameters:        files["parameters"] = (None, json.dumps(parameters))

    r = requests.post(f"{BASE}/api/reports/render", files=files)
    if not r.ok:
        try:    raise RuntimeError(r.json()["message"])
        except (ValueError, KeyError): r.raise_for_status()
    return r.content

public sealed class LeoJasperClient(HttpClient http) {

        string templateId,
        object data,

        string outputFormat,
        IDictionary<string, object>? parameters = null,
        CancellationToken ct = default) {

        form.Add(new StringContent(templateId), "templateId");

java.net.http multipart client (JDK 11+)

import java.net.URI;
import java.net.http.*;
import java.nio.charset.StandardCharsets;
import java.nio.file.*;
import java.util.*;

public class LeoJasperClient {
    private final HttpClient http = HttpClient.newHttpClient();
    private final String base;
    public LeoJasperClient(String base) { this.base = base; }

    public byte[] render(String templateId, String dataInlineOrJson,
                         String inputFormat, String outputFormat,
                         Map<String, Object> parameters) throws Exception {
        String boundary = "----leoJ" + System.nanoTime();
        var parts = new ArrayList<byte[]>();
        parts.add(field(boundary, "templateId",   templateId));
        parts.add(field(boundary, "dataInline",   dataInlineOrJson));
        parts.add(field(boundary, "inputFormat",  inputFormat));
        parts.add(field(boundary, "outputFormat", outputFormat));
        if (parameters != null) {
            parts.add(field(boundary, "parameters",
                            new com.fasterxml.jackson.databind.ObjectMapper()
                                .writeValueAsString(parameters)));
        }
        parts.add(("--" + boundary + "--\r\n").getBytes(StandardCharsets.UTF_8));
        byte[] body = concat(parts);

        HttpRequest req = HttpRequest.newBuilder(URI.create(base + "/api/reports/render"))
            .header("Content-Type", "multipart/form-data; boundary=" + boundary)
            .POST(HttpRequest.BodyPublishers.ofByteArray(body))
            .build();
        HttpResponse<byte[]> res = http.send(req, HttpResponse.BodyHandlers.ofByteArray());
        if (res.statusCode() / 100 != 2) {
            throw new RuntimeException(new String(res.body(), StandardCharsets.UTF_8));
        }
        return res.body();
    }

    private static byte[] field(String b, String name, String value) {
        return ("--" + b + "\r\n" +
                "Content-Disposition: form-data; name=\"" + name + "\"\r\n\r\n" +
                value + "\r\n").getBytes(StandardCharsets.UTF_8);
    }
    private static byte[] concat(List<byte[]> chunks) {
        int n = chunks.stream().mapToInt(c -> c.length).sum();
        byte[] out = new byte[n]; int off = 0;
        for (byte[] c : chunks) { System.arraycopy(c, 0, out, off, c.length); off += c.length; }
        return out;
    }
}

// use
var client = new LeoJasperClient("http://localhost:8080");
byte[] pdf = client.render(
    "sample",
    "[{\"name\":\"Alpha\",\"qty\":3}]",
    "json", "pdf",
    Map.of());
Files.write(Paths.get("report.pdf"), pdf);

Parameter type coercion ¶

Before filling, the service walks the JRXML's declared <parameter/> elements and coerces each supplied value to the declared Java type. Producers that send numeric data with surprising precision (a long-form decimal, a bare integer, a BigDecimal) don't have to match the JRXML's class attribute exactly — but a real type mismatch (sending a String when the report expects a number) still fails fast with a clear 400, not a cryptic JR cast exception during evaluation.

Rules applied

DeclaredSuppliedOutcome
Double / Float any Number (Integer, Long, BigDecimal, …) silently converted via .doubleValue() / .floatValue()
Integer / Long / Short / Byte any Number silently converted via .intValue() / .longValue() / …
BigDecimal any Number new BigDecimal(n.toString())
BigInteger any Number BigInteger.valueOf(n.longValue())
String anything non-null String.valueOf(v) — no error
Boolean "true" / "false" string Boolean.parseBoolean
Number of any kind String "5" HTTP 400 with the actual type mismatch
Anything else (Date, custom POJO) non-matching HTTP 400, clear message

Concrete examples

# JRXML declares: <parameter name="totals.subTotal" class="java.lang.Double"/>

{"totals.subTotal": 46}             ✓ Integer → Double
{"totals.subTotal": 46.0}           ✓ Double → Double
{"totals.subTotal": 46.000000000}   ✓ Double (still fits)
{"totals.subTotal": 0.0000000000000000000000000000}
                                    ✓ BigDecimal → Double (precision clipped — acceptable)
{"totals.subTotal": null}           ✓ stays null (JR renders blank)

{"totals.subTotal": "46"}           ✗ HTTP 400: String→Number rejected

Error shape for type mismatches

{
  "status":  400,
  "error":   "Bad Request",
  "message": "Parameter 'totals.subTotal' is declared as java.lang.Double
              but received java.lang.String (\"46\"). Only Number→Number
              and Anything→String coercions are applied automatically;
              fix the source data to send the correct type.",
  "timestamp": "2026-05-12T09:00:00Z"
}

How to fix a String→Number rejection in your client

  • JavaScript: use Number(value) when building the JSON. JSON.stringify({"qty": Number("2")}) emits {"qty":2} as a JSON number, not a string.
  • C#: bind the JSON property to double / decimal in your DTO, not string. The default serializer emits numbers correctly.
  • Java: same — typed POJO field, not String.
  • Python: pass int / float to json.dumps. {"qty": int(qty)} emits a JSON number.

Image resolution ¶

JRXML <imageExpression> values are intercepted by a per-company image resolver during fill, so reports keep rendering even when a logo is missing or a remote URL is unreachable.

Resolution order

If the expression evaluates to…Behaviour
http://… or https://… Fetched with a 5 s connect + 5 s read timeout. On any failure (DNS, 4xx/5xx, timeout, malformed response) the placeholder is substituted — the report still renders.
data:image/…;base64,… Decoded natively by JR.
A filename like "logo.png" or "MTC_LOGO" Basename looked up under your company's images/ bucket first, then companyDir/, then the other buckets. First hit wins.
A leading-slash path like "/Files/Branding/Logo.png" The leading directory is stripped; basename (Logo.png) looked up exactly like a plain filename. Useful when an ERP embeds absolute paths from its own filesystem.
Anything else (file not found, URL fetch failed) A generated 300×80 grey PNG labelled "Resource not found" is rendered into the slot. The fill does not fail.

How to upload an image so the resolver finds it

curl -X POST "http://localhost:8080/api/assets/images/logo.png" \
     -H "X-Key-Id: $KEY_ID" -H "X-Key-Secret: $SECRET" \
     -F "file=@./logo.png"

# JRXML then references it by either:
<imageExpression><![CDATA["logo.png"]]></imageExpression>
<imageExpression><![CDATA["logo"]]></imageExpression>
<imageExpression><![CDATA[$P{COMPANY_LOGO}]]></imageExpression>  # with COMPANY_LOGO="logo.png"

What's NOT intercepted

The resolver only fires for image-like paths — URLs, paths with image extensions (.png .jpg .jpeg .gif .svg .webp .bmp .tif .tiff .ico), or extension-less identifiers like "MTC_LOGO". Subreports (.jasper / .jrxml), properties files, and fonts flow through to JR's default resolvers untouched.

Making a specific image fatal on failure

Sometimes you want a missing image to abort the render (e.g. a regulatory seal that must always render). Set onErrorType="Error" on that one <image> element in the JRXML — it overrides the per-fill resolver's silent fallback: 

<image onErrorType="Error">
  <reportElement x="0" y="0" width="100" height="60"/>
  <imageExpression><![CDATA["regulatory-seal.png"]]></imageExpression>
</image>

Per-fill: no LicenseContext = no resolver

The custom resolver requires an authenticated request (so we know which company's images/ to search). CLI runs or callers without license headers get JR's default behaviour, which throws on missing images. This only matters if you bypass the standard /api/reports/render route.

Error model ¶

Every 4xx/5xx response is JSON in this shape:

{
  "status":    400,
  "error":     "Bad Request",
  "message":   "Unknown templateId: invoice",
  "timestamp": "2026-05-08T14:42:00Z"
}
CodeTypical cause
400 Wrong templateId, malformed parameters JSON, unsupported format, JRXML schema validation, missing required field, parameter type mismatch the auto-coercer can't bridge (e.g. supplying a JSON string for a numeric parameter — see Parameter type coercion), or an attempt to set assetsRoot to a folder that already contains a vault.
404 Job id / template id not found, or stray static path.
409 Calling /jobs/{id}/result before the job is COMPLETED.
413 Multipart payload exceeds the server limit (default 50 MB; configurable in application.yml).
500 JasperReports parsing/rendering failure inside the engine. message carries the JR error verbatim — usually a missing field, type mismatch, or band-overflow.

Tips & gotchas ¶

  • Numbers in JSON must be JSON numbers. Emitting "qty":"2" (a string) for a JRXML java.lang.Integer field is rejected with HTTP 400. The auto-coercer handles Number→Number (Integer 0 → Double 0.0, BigDecimal → Double, etc.) but not String→Number. Fix the producer to emit a real JSON number, not a quoted string. See Parameter type coercion for the full table.
  • Images get a placeholder on miss, never an error. If a logo isn't uploaded or a URL is down, the slot renders a 300×80 grey "Resource not found" PNG and the report continues. Upload company images via POST /api/assets/images/<name>, reference them by filename in the JRXML, or by $P{NAME} with a parameter value of the filename. See Image resolution.
  • JSON shape. The default JSON data source iterates the root if it's an array ([{…},{…}]). Wrap an array inside an object only if your JRXML sets <queryString language="json">items</queryString>.
  • CSV. First line must be the header row. Header names must match the JRXML <field name="…"/> declarations.
  • Grouping. Sort the data by the group field before sending. The server does not sort.
  • Field types. The JRXML class attribute is strict — java.lang.Integer rejects non-numeric strings. Use java.lang.String when in doubt and convert in expressions.
  • Currency / dates. Format on the server side via the pattern attribute on the <textField>, not on the client. e.g. pattern="$#,##0.00" or pattern="yyyy-MM-dd".
  • Async vs sync. Anything that takes > ~30 s should go through /render-async; browsers and proxies time out longer sync requests routinely.
  • Binary in JS. Always read the success body as a Blob. For axios, set responseType: "blob". Never JSON-parse the success body — only the error body is JSON.
  • CORS in production. The dev config is wide-open (allowedOriginPatterns="*"). Lock it to your real frontend origin before shipping.
  • Templates hot-reload. Drop a new .jrxml into the server's templates/ directory and the next render picks it up — the cache invalidates on mtime change.
